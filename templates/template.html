O que é o Flask

É um micro-framework, escrito em Python, para programar voltado a web.

  • O que quer dizer micro-framework ?
    • Engine simples e extensível
    • Não toma decisões por você
    • Leve!
    • Dá para criar uma app web com somente 1 arquivo!
  • O que vem junto com o Flask ?
    • servidor de desenvolvimento com debugger avançado
    • suporte a testes unitários
    • engine de request/response compatível com RESTFull
    • engine de templates simples - Jinja 2
    • 100% WSGI 1.0
    • unicode

Werkzeug

The Python WSGI Utility Library

Um conjunto de ferramentas WSGI, é utilizada principalmente no Flask, porém pode ser utilizada como uma lib em outras aplicações também.

Contém uma série de utilidades para um sistema web:

  • Sistema de roteamento de URLs
  • Debugador facilitado, com Javascript e micro-terminal (fácil de usar o pdb)
  • Objetos de Response/Request fáceis de utilizar
  • Controle de Cache
  • Envio de arquivos
  • Sessão em modo seguro via Cookies ou normal

Jinja 2

A full featured template engine for Python

Foi inspirada no motor de templates do Django, porém foi aprimorada com muitas ferramentas.

Algumas das suas características são:

  • Herança de templates
  • Compilação Just-in-time (compila em tempo de execução para bytecodes)
  • Integrada com o Flask/Werkzeug, então é fácil de debugar
  • Grande performance

Quem usa: Instagr.am, Sourceforge, Mozilla, etc...

Plugins do Flask

O Flask tem um número bastante grande de plugins disponíveis, com eles você pode montar a sua aplicação do jeito que gostar e necessitar, mantendo todo o controle do seu funcionamento e arquitetura.

  • flask-admin (Cria uma interface de administração)
  • flask-sqlalchemy (Suporte à biblioteca SQLAlchemy)
  • flask-flatpages (Provê páginas estáticas via Markdown)
  • flask-oauth (Suporte p/ autenticação OAuth)
  • flask-login (Suporte completo para servir login em sua aplicação)
  • Flask-Restless (Geração de RestFULL Apis através do SQLAlchemy)
  • flask-themes (Suporte para temas na sua aplicação)
  • flask-uploads (Ferramentas de auxilio para trabalhar com envio de arquivos)
  • flask-wtf (Integração para WTForms)
  • Flask-PyMongo (Conexão ao MongoDB via PyMongo)
  • e muitos outros...

Banco NoSQL

from flask import Flask
from flask.ext.pymongo import PyMongo

app = Flask(__name__)
mongo = PyMongo(app)

@app.route('/')
def home_page():
    online_users = mongo.db.users.find({'online': True})
    return render_template('index.html',
        online_users=online_users)

@app.route('/uploads/', methods=['POST'])
def save_upload(filename):
    mongo.save_file(filename, request.files['file'])
    return redirect(url_for('get_upload', filename=filename))

class WindowView(MethodView):

Blueprints

O Flask usa o conceito de blueprint para criar seus componentes "plugáveis", obedecendo os padrões de uma aplicação através da aplicação.

blueprints:

  • tem estrutura parecida com a de uma app Flask
  • simplificam e organizam a ampliação de uma aplicação
  • são registrados para urls específicas
  • podem ser registrador para várias urls diferentes na mesma app
  • é basicamente um módulo python dentro da nossa app

Além disto:

  • modularize sua aplicação
  • separe as partes (views, models, etc..) e importe-as

Mais de Flask

Json Suport
Método "jsonify" retorna já um json dos objetos em um objeto Response, com mimetype "application/json"
Sinais
São "eventos" que podemos conectar para receber ou tratar dados e ações
Views Plugáveis
Estas views são baseadas em classes e não em métodos conforme vemos no padrão, assim podemos utilizar OO nas views tb
Message Flashing
É um meio de empilhar mensagens que podem ser desempilhadas e mostradas no seus templates, não necessitando de maior controle
Logs
Sistema de log padrão Python, compatível com arquivo, emails, stdout, etc...
